# cpp_onion ^.^

Генератор vanity-адресов Tor v3 (`.onion`). Ищет адрес, начинающийся с заданного префикса.

## Сборка

CPU (release):

```sh
cmake --build --preset release
# бинарь: ./build/release/src/cli/onion
```

CUDA (GPU, через preset):

```sh
cmake --preset cuda && cmake --build --preset cuda
# бинарь: ./build/cuda/src/cli/onion
```

## Базовый поиск (CPU)

Найти один адрес с префиксом `synx`, ключи положить в `./keys`:

```sh
./build/release/src/cli/onion synx -o ./keys
```

## CUDA поиск (быстро, GPU)

То же самое, но на видеокарте:

```sh
./build/cuda/src/cli/onion synx --engine cuda -o ./keys
```

## CPU + GPU вместе (максимум скорости на одной машине)

`--engine cpu+gpu` гоняет процессорный и видео-движок **одновременно** на
непересекающихся участках поиска — это самый быстрый режим (~415 M/s = вся
видяха + все ядра CPU, +6.6% к чистой GPU). По умолчанию берёт все логические
ядра под CPU-движок; видео-поток спит на синхронизации, а не жжёт ядро, поэтому
процессор не отнимает скорость у GPU. Любой найденный ключ всё так же проходит
проверку `io::verify` перед записью.

```sh
./build/cuda/src/cli/onion synx --engine cpu+gpu -o ./keys
# бенч объединённой скорости:
./build/cuda/src/cli/onion zzzzzzzzzzzzzzzz --engine cpu+gpu --bench 10
```

## Несколько адресов

Собрать 5 разных адресов с префиксом `abc` (CUDA):

```sh
./build/cuda/src/cli/onion abc --engine cuda -n 5 -o ./keys
```

## Бесконечный поиск

Брутить префикс, пока не остановишь вручную (Ctrl+C); `-n 0` = без ограничения:

```sh
./build/cuda/src/cli/onion abc --engine cuda -n 0 -o ./keys
```

## CPU в несколько потоков

15 адресов в 4 потока:

```sh
./build/release/src/cli/onion abc -n 15 -t 4 -o ./keys
```

## Несколько префиксов сразу

Подойдёт любой из перечисленных (`abc` ИЛИ `xyz23`):

```sh
./build/cuda/src/cli/onion abc xyz23 --engine cuda -o ./keys
```

## Бенчмарк скорости

Гонять 10 секунд против заведомо ненаходимого префикса и вывести keys/s:

```sh
./build/cuda/src/cli/onion zzzzzzzzzzzzzzzz --engine cuda --bench 10
```

## Тихий режим и куда падают ключи

`-q` / `--quiet` — убрать баннер и прогресс, печатать только строку с найденным адресом:

```sh
./build/cuda/src/cli/onion synx --engine cuda -q -o ./keys
```

На каждый найденный адрес создаётся папка вида:

```
<out>/<полный-адрес>.onion/
├── hostname               # сам адрес, права 600
├── hs_ed25519_secret_key  # секретный ключ, права 600
└── hs_ed25519_public_key  # публичный ключ, права 600
```

Папка создаётся с правами 700, файлы — 600 (формат полностью совместим с Tor).

Подключение к Tor: скопируй папку в каталог hidden service и в `torrc` пропиши:

```
HiddenServiceDir /var/lib/tor/<полный-адрес>.onion/
HiddenServicePort 80 127.0.0.1:8080
```

## Флаги

| Флаг | Назначение |
|------|------------|
| `--engine` | движок: `incremental` (по умолчанию), `naive`, `cuda` (GPU, нужна CUDA-сборка) |
| `-n`, `--count` | сколько адресов найти; `0` = бесконечно (Ctrl+C для остановки) |
| `-t`, `--threads` | число потоков (только CPU) |
| `-o`, `--out` | каталог для ключей |
| `-q`, `--quiet` | без прогресса, только найденные адреса |
| `--bench` | бенчмарк: N секунд против ненаходимого префикса, вывести keys/s |
| `--simd` | AVX2 4-wide движок: `auto` (по умолч.), `on`, `off` (CPU) |
| `--batch` | размер батча incremental-движка (по умолч. 1024) |

## Сколько ждать

В среднем нужно `32^L` попыток, где `L` — длина префикса (32 символа алфавита base32 на позицию). На GTX 1650 ориентировочно:

| Длина | Время |
|-------|-------|
| ≤ 6 символов | секунды |
| 7 символов | минуты |
| 8 символов | ~час |

Каждый адрес длиннее тянет за собой ~32x времени.

## Проверка

Перед записью на диск каждый найденный ключ перепроверяется через libsodium — на диск попадают только заведомо корректные ключи.
